##############################################################################
##  未来 UI: 交互式 mech interp 浏览器
##############################################################################

==== 用户交互流程 ====

  1. 顶部: 输入 prompt → 自动 forward → 显示模型生成 token 序列
  2. 用户点击生成结果中的某个 token
       (例如点击 " Google", 系统进入"分析 ' Google' 怎么生成的"模式)
  3. 左侧面板: 显示该 token last_pos 的 35 层 hidden state 时间线
       - 35 个色块/小图, 颜色 = top-1 类别
       - 悬停显示该层 logit lens top-1
       - 高亮 phase transition 层 (如 L24, L26, L28)
  4. 点击某一层 (如 L25)
  5. 中间面板: 显示 L25 的 treemap (复用 plot_hidden_treemap 风格)
  6. 点击 treemap 中某个 token cell 或 类别 strip
       (例如点击 'Google 多语言' strip 或 ' OpenAI' cell)
  7. 右侧面板: 显示 ±贡献该 token 的 top-N 神经元 list
       - 神经元 ID (如 L28#2406)
       - contribution 数值 (per-neuron DLA)
       - 该神经元 W_down 列向量的 top-10 vocab 投影 (神经元的"语义指向")
       - 该神经元的激活值
       - [可选] "clamp 它看效果" 按钮 (单神经元 ablation 实时试)

==== 实施分阶段 ====

  Phase 1 — Streamlit MVP (1-2 天)
    技术: streamlit + plotly + 现有 plot 脚本
    限制: 不要实时点击, 用 dropdown/slider 选 layer / token
    每次切换重新生成图
    够用来 demo 给别人看核心机制

  Phase 2 — Full UI (1-2 周)
    技术: FastAPI backend + Svelte/React frontend + WebSocket
    模型常驻内存, 多 tab 共享同一个加载的模型
    每个 prompt 单次 forward 缓存所有层 hidden + 所有位置
    D3.js treemap (可点击 cell)
    实时神经元探查面板

  Phase 3 — 集成 SAE (要先训出 SAE)
    每层 hidden 自动用 SAE 分解为 features
    treemap 显示 features 而非 raw token (信息更干净)
    点击 feature → 显示该 feature 的训练样本 (像 Anthropic 的 demo)

==== 数据 / 后端架构 ====

  - 单进程加载 Gemma 4 E2B (BF16 ~9GB GPU)
  - 每个 prompt 跑一次 forward, capture:
      hidden_states_per_layer    [n_layers, T, hidden]
      attn_outputs_per_layer     [n_layers, T, hidden]
      mlp_outputs_per_layer      [n_layers, T, hidden]
      mlp_intermediates          [n_layers, T, intermediate_L]
  - 总缓存约 200MB-1GB / prompt
  - LRU cache 最多缓存 N 个 prompt

  - API 端点:
      POST /forward          { prompt }              → returns 生成结果
      GET  /token/{i}/layers                         → 35 层 logit lens 摘要
      GET  /token/{i}/layer/{L}/treemap              → 该层 top-K 分类
      GET  /token/{i}/layer/{L}/neuron_attribution   → per-neuron DLA
      POST /clamp                { layer, neuron, value, prompt }
                                                     → 试干预看效果

==== 命名 ====

  暂定: "Gemma Lens" 或 "amr_lens" 或就叫 "identity_swap_ui"
